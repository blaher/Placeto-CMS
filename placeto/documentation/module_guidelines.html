Here at BlaherTech.org, we are very picky. In order to publish a module, we prefer you follow a few guidelines when writing your code. The majority of us come from an C background who all like the poetry of Java, but dislike the JVM. So try and write your code as if you were writing a Java poem.

  • Use PhpDoc, always and everywhere.
  • We use NetBeans to develope Placeto, use what you want, but we will provide you with NetBean project files.
  • We only accept open-source, specified by either the OSI or FSF foundations.
  • Make sure to include the GPL statement, if using GPL.
  • Do not leave any class function scope's unset (e.g. private, public, protected), same goes for class variables.
  • Unset any variables when you're done with them.
  • Programming style must be readable, we also prefer using the Allman style.
  • Like drinking water, using comments is a healthy part of Placeto's diet.
  • Use PDO to get SQL feedback, we provide you with an already running object.
  • Do not directly concatinate anything in to SQL, append them with PDO Object::execute.
  • Please use the security tools we provide you, whenever possible.
  • Remember to give credit to yourself and your staff, where it can be applied.
